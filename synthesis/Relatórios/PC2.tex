\begin{resumo}
 Este relatório apresenta o desenvolvimento de uma unidade de processamento
 para um processador RISC-V multiciclo.
 O foco principal foi implementar a lógica de execução de algumas instruções do módulo RV32I, utilizando SystemVerilog. 
 A unidade de controle foi desconsiderada neste estágio, 
 e o objetivo foi validar a integração entre ALU, banco de registradores, multiplexadores e lógica de PC. 

 \noindent
 \textbf{Palavras-chaves}: RISC-V. SystemVerilog. Processador. Multiciclo.
\end{resumo}

\chapter{Introdução}
Este relatório foi desenvolvido como parte da disciplina de 
Laboratório de Sistemas Computacionais: Arquitetura e Organização de Computadores.
O objetivo central é implementar uma unidade de processamento para um processador RISC-V 
com abordagem multiciclo. A arquitetura foi projetada para ser modular e extensível, 
facilitando futuras integrações com unidade de controle, periféricos e outros módulos para o conjunto de intruções.

O projeto é baseado em um diagrama de blocos que representa os principais componentes de um processador: 
banco de registradores, \emph{ALU}, multiplexadores, registradores intermediários (como A, B, ALUOut), 
unidade de memória e extensores de sinal. O design foi desenvolvido em SystemVerilog.

\section{Geral}
Implementar uma unidade de processamento funcional baseada na arquitetura RISC-V multiciclo, 
utilizando SystemVerilog para as instruções principais do conjunto RV32I.
\section{Específico}

\begin{itemize}
\item Desenvolver e interconectar os módulos da ALU, banco de registradores e demais componentes conforme o diagrama de blocos fornecido.

\item Possibilitar a execução correta de instruções como ADD, SUB, AND, LW, SW, BEQ, JAL, entre outras.

\end{itemize}

\chapter{Fundamentação Teórica}
A implementação de um processador RISC-V multiciclo envolve conhecimentos consolidados em organização e arquitetura de computadores, 
especialmente no que diz respeito ao fluxo de dados e controle de instruções. Segundo Patterson e Hennessy (2014), 
a abordagem multiciclo divide a execução de cada instrução em etapas distintas 
— como busca de instrução (\emph{IF}), decodificação (\emph{ID}), execução (\emph{EX}), acesso à memória (\emph{MEM}) e escrita no registrador (\emph{WB}) — 
permitindo a reutilização de componentes em diferentes ciclos do clock. Essa estratégia é fundamental para otimizar a utilização do hardware e facilitar o controle de cada etapa de execução.

A arquitetura RISC-V, proposta inicialmente por Waterman e Asanović (2016), é uma \emph{ISA}(\emph{Instruction Set Architecture}) de código aberto que prioriza simplicidade, modularidade e extensibilidade. 
O conjunto base de instruções RV32I define operações aritméticas, lógicas, de controle de fluxo e acesso à memória, suficientes para a implementação de um processador funcional. 

O funcionamento correto da unidade de processamento depende da cooperação entre diversos módulos. A \emph{ALU} (\emph{Arithmetic Logic Unit}), 
realiza operações como adição, subtração, operações lógicas e comparações, é um dos principais componentes e deve ser projetada para suportar os sinais de operação definidos no conjunto RV32I. 
O banco de registradores, com 32 registradores de 32 bits, armazena e fornece operandos às instruções. O controle de leitura e escrita nesse banco é feito por meio de sinais sincronizados com o clock.

Outros elementos fundamentais incluem os registradores intermediários que guardam certos dados entre os ciclos,
os multiplexadores, responsáveis pela seleção de dados entre diferentes fontes, e os extensores de sinal, 
que são usados para adaptar imediatos de 12 ou 20 bits para o tamanho padrão de 32 bits.
